# Evals

This folder covers the evaluation half of the workshop. Datasets and
experiments use the unified **Arize Python client** (`arize` v8+); the
continuous **online evaluator** uses the **`ax` CLI**.

All scripts read credentials from the environment:

```bash
export OPENAI_API_KEY="sk-..."
export ARIZE_SPACE_ID="U3BhY2U6..."
export ARIZE_API_KEY="ak-..."     # Settings -> Space API Keys
```

Run them from the repo root (`python evals/<script>.py`).

## 1. Build the golden dataset

```bash
python evals/build_golden_dataset.py
```

Uploads `datasets/golden_support.json` (~16 hand-labeled support examples,
including edge cases and "should escalate" rows) as the `sunrise-support-golden`
dataset. View it in Arize under **Datasets**.

## 2. Offline evals

```bash
python evals/offline_evals.py
```

Runs the agent over the golden dataset as an experiment (`offline-baseline`),
scoring each row with four evaluators:

| Evaluator | Type | Checks |
|-----------|------|--------|
| `tool_selection` | code | the expected tool was called |
| `escalation_appropriate` | code | escalation happened iff expected |
| `correctness` | LLM judge | reply matches the expected answer |
| `groundedness` | LLM judge | reply is supported by the retrieved context |

Open the experiment in Arize to see per-example scores.

## 3. Experiment: compare two variants

```bash
python evals/run_experiment.py
```

Runs two experiments over the same dataset — `retrieval-on` (full agent) and
`retrieval-off` (retrieval tool removed). Open `sunrise-support-golden` in Arize
and use the **Experiment Comparison** view. Expect groundedness and correctness
on policy questions to drop noticeably without retrieval — that contrast shows
why retrieval matters.

## 4. Curate a dataset from real traces

```bash
python evals/curate_from_traces.py --hours 24 --limit 100
```

Pulls recent spans from the `arize-singapore-workshop` project (the traffic
generated by chatting via `app.py` or the notebook) and builds the
`sunrise-support-from-traces` dataset. From there you can iterate an eval
template on real data and route rows through a labeling queue for ground truth.

## 5. Deploy a continuous online evaluator (ax CLI)

```bash
pip install arize-ax-cli
python evals/setup_online_eval.py
```

Creates an OpenAI AI integration (from `OPENAI_API_KEY`), a template
(LLM-as-a-judge) evaluator (`resolved` / `not_resolved`), and a **continuous,
trace-level** evaluation task on the project, then triggers a run. New traces
are scored automatically. Watch results under the project's **Evals** tab.

### What `setup_online_eval.py` does, step by step

```bash
# 1) AI integration that powers the LLM judge
ax ai-integrations create --name workshop-openai --provider openAI \
  --api-key "$OPENAI_API_KEY" --enable-default-models --function-calling-enabled

# 2) Template (LLM-as-a-judge) evaluator
ax evaluators create-template-evaluator --name support-resolution \
  --space "$ARIZE_SPACE_ID" --commit-message "initial version" \
  --template-name resolution --template '<judge prompt with {{input}}/{{output}}>' \
  --ai-integration-id <AI_INTEGRATION_ID> --model-name gpt-4o-mini \
  --classification-choices '{"resolved": 1, "not_resolved": 0}' \
  --include-explanation --data-granularity trace

# 3) Attach as a CONTINUOUS online task on the project
ax tasks create-evaluation --name support-resolution-online \
  --task-type template_evaluation \
  --evaluators '[{"evaluator_id": "<EVALUATOR_ID>", "column_mappings": {"input": "attributes.input.value", "output": "attributes.output.value"}}]' \
  --project arize-singapore-workshop --space "$ARIZE_SPACE_ID" \
  --is-continuous --sampling-rate 1.0

# 4) Run it now (continuous tasks also pick up new traces automatically)
ax tasks trigger-run support-resolution-online
ax tasks list-runs support-resolution-online
```

## The full loop

`build` (agent) → `trace` (notebook / app.py) → `offline eval` (offline_evals.py) +
`experiment` (run_experiment.py) → **generate real traces** (Gradio chat) →
`curate` (curate_from_traces.py) → `online eval` (setup_online_eval.py).
